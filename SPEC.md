# QR Parking Pay — Спецификация приложения

## Описание

Демо-приложение для оплаты парковки через сканирование QR-кода.
Реализовано как одностраничное мобильное веб-приложение (SPA).

---

## Стек

| Технология | Версия |
|---|---|
| React | 19 |
| Vite | 8 |
| html5-qrcode | latest |
| CSS | чистый CSS, без UI-библиотек |

---

## Интерфейс

### Страница сканирования (единственная страница)

Страница разделена на два состояния: **ожидание** и **сканирование**.

#### Состояние «Ожидание»

- Отображается кнопка **«Сканировать QR»**.
- Камера не активна.

#### Состояние «Сканирование»

- Активируется после нажатия кнопки «Сканировать QR».
- На экране отображается трансляция с камеры.
- При успешном считывании QR-кода устройство воспроизводит тактильный отклик через `navigator.vibrate()` (если поддерживается браузером).
- Доступна кнопка **«Включить фонарь»** (отображается только если устройство поддерживает управление вспышкой через `ImageCapture API`; при повторном нажатии — выключить фонарь).
- Доступна кнопка **«Отмена»** для возврата в состояние «Ожидание».

---

## Функциональность сканера

- Для сканирования используется библиотека **[html5-qrcode](https://github.com/mebjas/html5-qrcode)**.
- Сканер активируется на заднюю камеру устройства (`facingMode: environment`).
- Сканирование происходит непрерывно — без необходимости делать снимок вручную.
- После успешного считывания QR-кода сканер останавливается автоматически.

---

## Визуальный прицел

- Поверх трансляции камеры отображается визуальная рамка-прицел (оверлей).
- Помогает пользователю правильно наводить камеру на QR-код.
- Реализуется через CSS-наложение — без влияния на логику сканирования.

---

## Валидация QR-кода

- QR-код считается **валидным**, если содержит корректный URL по схеме `http://` или `https://`.
- Любое другое содержимое (текст, невалидный URL) считается **невалидным**.

---

## Сценарии после сканирования

### Валидный QR (URL)

1. Камера останавливается.
2. Отображается диалог подтверждения с текстом вида:
   > Перейти по ссылке для оплаты парковки?
   > `<URL>`
3. Кнопки: **«Перейти»** и **«Отмена»**.

**Если пользователь нажал «Перейти»:**
- URL открывается в новой вкладке (`target="_blank"`, `rel="noopener noreferrer"`).
- Камера остаётся закрытой.
- Приложение возвращается в состояние «Ожидание».

**Если пользователь нажал «Отмена»:**
- Диалог закрывается.
- Камера возобновляет сканирование.

### Невалидный QR

1. Камера не останавливается.
2. Отображается уведомление:
   > QR-код не распознан как ссылка для оплаты.
3. Уведомление исчезает автоматически через 3 секунды.
4. Сканирование продолжается.

---

## Разрешения

- При первом запуске сканера браузер запрашивает разрешение на доступ к камере.
- Если разрешение отклонено — отображается сообщение об ошибке с инструкцией, как включить доступ в настройках браузера.

---

## Требования к устройству

- Приложение предназначено для использования на мобильных устройствах через браузер.
- Требуется HTTPS для работы с камерой (или `localhost` при разработке).
- Поддержка фонаря опциональна — кнопка не отображается, если устройство не поддерживает управление вспышкой.
