# QR Parking Pay — Спецификация приложения

## Описание

MVP-приложение для проверки суммы оплаты парковки.
Реализовано как одностраничное мобильное веб-приложение (SPA).

Базовый пользовательский сценарий:
1. Пользователь вводит номер автомобиля или парковочного талона.
2. Пользователь запускает проверку суммы.
3. Пользователь сканирует QR-код паркомата.
4. Приложение показывает результат: оплата не требуется или сумму к оплате.

## Интерфейс

### Единственная страница приложения

Структура страницы при запуске приложения:
- хидер (остается как есть);
- блок с текстом:
   > Для проверки суммы оплаты парковки введите номер автомобиля или парковочного талона, после отсканируйте QR-код паркомата. Если время бесплатной парковки истекло - приложение покажет сумму к оплате
- форма для ввода;
- футер (остается как есть).

Страница работает в двух основных состояниях:
1. **Форма** (стартовое состояние).
2. **Сканирование и результат** (после успешной валидации формы).

## Форма ввода

Форма содержит два таба, на каждом табе по одному полю ввода.

### Таб 1 (активный по умолчанию)

- Label: **«Номер автомобиля»**.
- Поле: строковый ввод.
- Placeholder: `0000 AA-7`.
- Поведение ввода: буквы автоматически переводятся в верхний регистр.
- Инициализация: при загрузке формы поле предзаполняется значением из `localStorage`, сохраненным при последнем использовании приложения.

### Таб 2

- Label: **«Парковочный талон»**.
- Поле: строковый ввод.
- Placeholder: `123456789`.
- Ограничение: `maxLength=9`.

### Кнопка отправки формы

- Под табами размещается кнопка **«Проверить сумму»**.

### Валидация формы

- Обязательным является только поле активного таба.
- При потере фокуса (`blur`) поля активного таба, если значение невалидно, показывается ошибка:
   > Обазательно для заполнения
- При попытке отправки формы (нажатие кнопки **«Проверить сумму»**) и невалидном активном поле показывается та же ошибка.
- Ошибка исчезает сразу после ввода значения в активное поле.

## Экран сканирования

Если форма валидна и пользователь нажимает кнопку **«Проверить сумму»**, открывается экран с превью камеры.

### Функциональность сканера

- Для сканирования используется библиотека **[html5-qrcode](https://github.com/mebjas/html5-qrcode)**.
- Сканер активируется на заднюю камеру устройства (`facingMode: environment`).
- Сканирование происходит непрерывно — без необходимости делать снимок вручную.
- При успешном считывании QR-кода устройство воспроизводит тактильный отклик через `navigator.vibrate()` (если поддерживается браузером).
- Доступна кнопка **«Включить фонарь»** (отображается только если устройство поддерживает управление вспышкой через `ImageCapture API`; при повторном нажатии — выключить фонарь).
- Доступна кнопка **«Отмена»** для возврата к форме.

### Визуальный прицел

- Поверх трансляции камеры отображается визуальная рамка-прицел (оверлей).
- Прицел помогает пользователю правильно наводить камеру на QR-код.
- Оверлей реализуется через CSS-наложение и не влияет на логику сканирования.

## Валидация QR-кода

- QR-код считается **валидным**, если содержит корректный URL по схеме `http://` или `https://`.
- Любое другое содержимое (текст, невалидный URL) считается **невалидным**.

---

## Сценарии после сканирования

### Валидный QR (URL)

1. Камера останавливается.
2. Отображается модальное окно с кнопкой **«Проверить сумму»**.
3. Нажатие кнопки инициирует имитацию API-запроса.

#### Имитация API-запроса

- Кнопка **«Проверить сумму»** переходит в состояние loading на 1.5 секунды.
- После 1.5 секунды показывается один из результатов:
   - в 2 из 3 случаев:
      > Время бесплатной парковки еще не закончилось. Оплата не требуется
   - в 1 из 3 случаев:
      > Время бесплатной парковки превышено на 65 минут. К оплате 1.42 руб.
- После отображения результата в модальном окне показывается кнопка **«Вернуться к форме»** для ручного возврата.

### Невалидный QR

1. Камера не останавливается.
2. Отображается уведомление:
    > QR-код не распознан как ссылка для оплаты.
3. Уведомление исчезает автоматически через 3 секунды.
4. Сканирование продолжается.

## Разрешения

- При первом запуске сканера браузер запрашивает разрешение на доступ к камере.
- Если разрешение отклонено, отображается сообщение об ошибке с инструкцией, как включить доступ в настройках браузера.

---

## Требования к устройству

- Приложение предназначено для использования на мобильных устройствах через браузер.
- Требуется HTTPS для работы с камерой (или `localhost` при разработке).
- Поддержка фонаря опциональна: кнопка не отображается, если устройство не поддерживает управление вспышкой.
